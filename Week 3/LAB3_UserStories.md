## กลุ่ม MintTea

### US-001: การเข้าสู่ระบบด้วยบัญชีมหาวิทยาลัย (SSO)

**As a** General user
**I want** เข้าสู่ระบบผ่านบัญชีของมหาวิทยาลัย (SSO)
**So that** ฉันสามารถเริ่มใช้งานระบบได้อย่างรวดเร็วและปลอดภัย

**Acceptance Criteria:** 
- ✅ สามารถ Loing ด้วย Student ID + Password ได้
- ✅ ระบบจะตรวจสอบความถูกต้องของ Student ID และรหัสผ่าน และอนุญาตให้ผู้ใช้เข้าสู่ระบบได้
- ✅ ผู้ใช้สามารถเข้าถึง Dashboard ได้ทันทีหลังจาก Login

---

### US-002: การสั่งอาหาร และ การจองโต๊ะ

**As a** General user  
**I want** ให้มีระบบการสั่งอาหาร การจองโต๊ะแยกกัน อย่างชัดเจน
**So that** ข้อมูลการสั่งอาหาร และ การจองโต๊ะของฉันถูกต้องและไม่สับสน

**Acceptance Criteria:**
- การจองโต๊ะอาหาร
  - ✅ ผู้ใช้สามารถเลือกวันที่, เวลา, จำนวนคน และ ร้านที่ต้องการที่ต้องการจองโต๊ะอาหารได้
  - ✅ ระบบตรวจสอบโต๊ะที่ว่างอยู่ และ บันทึกคำขอจองไว้
  - ✅ ระบบจะแสดง สถานะการจอง
- การสั่งอาหาร
  - ✅ ผู้ใช้สามารถ ระบุ รายการอาหาร , จำนวน และ ช่องทางการชำระเงินได้
  - ✅ ระบบจะทำการตรวจสอบเมนู , คำนวณราคา และ บันทึกคำสั่งไว้
  - ✅ ระบบจะแสดง หมายเลขคำสั่งซื้อ + สถานะเริ่มต้น

---

### US-003: ประวัติการจองโต๊ะ และ สั่งอาหาร

**As a** General user  
**I want** ที่จะสามารถทำให้ตรวจสอบ ประวัติการจองโต๊ะ สั่งอาหาร ได้ง่ายไม่ยุ่งยาก
**So that** ฉันสามารถดูรายละเอียดการจองและการสั่งอาหารที่เคยทำไว้ได้อย่างสะดวก

**Acceptance Criteria:**
- การตรวจสอบประวัติการจองโต๊ะอาหาร:
  - ✅ ผู้ใช้สามารถเข้าถึงหน้าประวัติการจองโต๊ะ
  - ✅ ระบบแสดงรายการการจองโต๊ะทั้งหมดที่ผู้ใช้ทำในอดีต
  - ✅ ผู้ใช้สามารถดูรายละเอียดการจองโต๊ะ (วันที่, เวลา, ร้านที่จอง, จำนวนคน, สถานะการจอง)
  - ✅ ระบบจะแสดงรายละเอียดการจองโต๊ะออกมา (วันที่, เวลา, ร้านที่จอง, จำนวนคน, สถานะการจอง)
- การตรวจสอบประวัติการสั่งอาหาร:
  - ✅ ผู้ใช้สามารถเข้าถึงหน้าประวัติการสั่งอาหาร\
  - ✅ ระบบแสดงรายการการสั่งอาหารทั้งหมดที่ผู้ใช้เคยทำ
  - ✅ ผู้ใช้สามารถดูรายละเอียดการสั่งอาหาร (รายการอาหาร, จำนวนเงิน, ชื่อร้าน, สถานะการสั่ง)
  - ✅ ระบบจะแสดงรายละเอียดการสั่งอาหารออกมา (รายการอาหาร, จำนวนเงิน, ชื่อร้าน, สถานะการสั่ง)

### US-004: ระบบการชำระเงิน

**As a** Student  
**I want** ฉันต้องการชำระเงินที่สะดวกสบาย
**So that** ที่ฉันจะมั่นใจได้ว่าการชำระเงินสำเร็จและสามารถดำเนินการสั่งอาหารต่อได้ทันที

**Acceptance Criteria:**
- ✅ รับ Payment ID จากคำสั่งซื้อที่อยู่ในสถานะรอชำระเงิน
- ✅ ตรวจสอบสถานะการชำระเงินผ่าน API ของ Payment Gateway
- ✅ แสดงผลลัพธ์การชำระเงิน
- ✅ อัปเดตสถานะคำสั่งซื้อตามผลลัพธ์

### US-005: ระบบให้คะแนนร้านอาหาร

**As a** Student  
**I want** ให้คะแนนกับร้านอาหารที่ฉันสั่งอาหาร
**So that** แสดงความรู้สึกและความคิดเห็นที่มีต่อร้าน

**Acceptance Criteria:**
- ✅ ผู้ใช้ต้องสามารถเลือกหรือระบุ Restaurant ID ของร้านที่เคยสั่งได้
- ✅ ระบบต้องอนุญาตให้ผู้ใช้ให้ คะแนนร้าน (เช่น 1–5 ดาว) ได้
- ✅ ระบบต้องรองรับการ พิมพ์ความคิดเห็น จากผู้ใช้
- ✅ เมื่อส่งรีวิว ระบบต้องบันทึกข้อมูลรีวิวสำเร็จและแสดงผลให้ผู้ใช้เห็น

### US-006: การสั่งซื้ออาหาร

**As a** Personnel  
**I want** ความสะดวกในการสั่งซื้ออาหาร
**So that** โดยไม่ต้องรอนาน หรือ สั่งล่วงหน้าได้มีรายการอาหาร ร้านค้าให้เลือกสั่งได้ง่ายๆ

**Acceptance Criteria :**
- ✅ มีหน้า แสดงลำดับคิวหรือคาดเดาเวลารออาหาร
- ✅ มีระบบสั่งซื้อ จองล่วงหน้า ได้
- ✅ มีรายการที่ดูง่าย
- ✅ มีปุ่มสำหรับติดตามการสั่งซื้อ

---

### US-007: Admin Dashboard สำหรับดูภาพรวม

**As a** System Admin 
**I want** เข้าถึง Admin Dashboard
**So that** ดูสถิติภาพรวมของทั้งมหาวิทยาลัย และ ใช้ในการติดตามและรายงานผลต่อผู้บริหาร

**Acceptance Criteria:**
- ✅ ผู้ใช้ที่มีสิทธิ์ Admin เท่านั้นที่สามารถเข้าถึงหน้านี้ได้
- ✅ Dashboard ต้องแสดงผลรวมการสั่งอาหาร จองโต๊ะ และรีวิวร้านอาหารของทั้งมหาวิทยาลัยแบบ Real-time
- ✅ ต้องมีกราฟแสดงจำนวนผู้ใช้งาน (Active Users) และสถิติทั้งหมด
- ✅ สามารถ Export ข้อมูลสรุปในรูปแบบ CSV หรือ Excel ได้

## 3) วิธีใช้ชุด SRS + User Stories นี้ใน LAB

1. **ทำ Use Case Diagram**  
   - จาก FR และ User Stories ด้านบน ให้ นศ. ระบุ Use Cases เช่น  
     - UC-01 View Lab List  
     - UC-02 View Lab Details  
     - UC-03 Submit Lab  
     - UC-04 View My Lab Status  
     - UC-05 Create Lab (Instructor)  
     - UC-06 Grade Submission  
   - วาด diagram (Student, Instructor, TA + System Boundary)

2. **เขียน Use Case Scenario** (อย่างน้อย 2–3 ตัว)  
   - แนะนำ:  
     - UC-03 Submit Lab (Student)  
     - UC-04 View My Lab Status (Student)  
     - UC-06 Grade Submission (Instructor)

3. **เชื่อมกับ LAB HTML/CSS (Week 3)**  
   - แปลง User Stories ที่เกี่ยวกับ Student เป็นหน้าเว็บ เช่น:
     - US-01 + US-02 → Section “Upcoming Labs” และตัวอย่าง Lab Detail บน `index.html`
     - US-03 + US-05 → ปุ่ม/section “Submit Lab” และแสดงข้อความ/สถานะ Late
     - US-04 → หน้า `my-labs.html` ที่มีตาราง/การ์ดแสดงสถานะ LAB
   - ใช้ NFR (Usability / Layout) เป็น guideline ในการออกแบบ HTML/CSS
